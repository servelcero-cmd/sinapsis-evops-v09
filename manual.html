Manual Interactivo — EV Ops (Santiago 20)

Este manual está pensado para que la demo salga impecable en reunión: qué hacer, qué mostrar y qué decir. Todo es SIMULATED (sin datos sensibles).

Panel de control

Estado: listo.

1) Inicio rápido (90 s)

Objetivo: entrar a EV Ops, simular cambios y crear 1 ticket.

Abrir la demo y entrar a ⚡ EV Ops Carga.
Activar Auto (si existe) o apretar Simular actualización 2 veces.
Seleccionar una estación “offline / cola alta” y Crear ticket.
Ir a Proyectos y mostrar el ticket creado con estado.

2) Recorrido guiado (7 min) — guion sugerido

0:00–1:00 “Hoy el nudo son puntos ciegos en carga. Sinapsis convierte operación en decisiones con trazabilidad.”

1:00–3:00 Mostrar EV Ops: estados, cola, disponibilidad, “casi tiempo real” (simulación).

3:00–5:00 KPI: uptime, disponibilidad, fuera de servicio, cola total + tendencia.

5:00–6:30 Ticket 1‑clic: incidente → responsable → evidencia → cierre (adiós Excel).

6:30–7:00 Cierre: “Piloto 4 semanas: 20 puntos → 50 reales con datos Enel + coaching de adopción.”

3) EV Ops: qué mirar (y por qué importa)

Disponibilidad: libre/ocupado/offline/cola.
Ejemplo: “Si está offline, no es marketing: es operación”.
Dinámica: cambios por minuto/turno.
Ejemplo: simular 2 updates y mostrar que el sistema “respira”.
Prioridad: dónde actuar primero.
Ejemplo: “cola alta + comuna clave” → ticket inmediato.

4) Tickets y seguimiento (anti‑Excel)

La promesa no es “más datos”: es acciones trazables (responsable, fecha, evidencia, cierre).

Crear ticket desde EV Ops.
Ejemplo: “Estación X: offline” → tarea “verificar energía / reset / visita”.
Estados (pendiente/en curso/resuelto).
Ejemplo: en Q&A, mostrar el cambio de estado en vivo.

5) KPIs clave (para directorio)

Uptime: % del tiempo disponible.
Ejemplo: meta piloto: +X puntos vs baseline.
Ocupación / cola: presión de demanda por zona.
Ejemplo: detectar horas punta para comunicación y re‑balance.
SLA de incidentes: tiempo a resolución.
Ejemplo: “80% resuelto < 24h” y evidencia.

6) Checklist pre‑reunión (2 min)

Internet OK (CDNs cargan React/Chart/jsPDF).
Ejemplo: abre una vez antes para cachear.
EV Ops visible y responde a “Simular”.
Ejemplo: 2 updates seguidos sin recargar.
Crear ticket funciona y aparece en Proyectos.
Ejemplo: ticket “Offline” con responsable.
Export/Import JSON listo (por si cambias de máquina).
Ejemplo: exporta, recarga, importa.

7) Publicación en URL (gratis, 3 minutos)

Opción A — Netlify Drop: crea cuenta → arrastra una carpeta con manual + demo → te entrega URL.

Opción B — GitHub Pages: repo nuevo → subes ambos HTML → Settings → Pages → Deploy.

Nota: yo no puedo subirlo por ti desde acá, pero el paquete viene listo para “arrastrar y soltar”.

8) Preguntas típicas (Q&A)

¿De dónde salen los datos?
Demo: simulados. Piloto: Enel define 20 puntos y habilita data mínima.
¿Qué reemplaza Excel?
Catastro único + tickets + KPIs. Excel pasa a ser input, no “sistema”.
¿Riesgos de privacidad?
Anonimización, roles, bitácora de decisiones IA. Sin PII para la demo.
